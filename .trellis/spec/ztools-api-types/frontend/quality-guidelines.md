# Quality Guidelines

> Quality expectations for `ztools-api-types` in this repo.

## Actual Rule

This package should currently be treated as synchronized type output, not as a normal frontend implementation area.

## Required Patterns

- Keep generated files and sync metadata consistent.
- Prefer changing the sync process over hand-maintaining divergent generated output.

## Forbidden Patterns

- Do not invent frontend architecture rules for a package that has no local frontend source checked out.
