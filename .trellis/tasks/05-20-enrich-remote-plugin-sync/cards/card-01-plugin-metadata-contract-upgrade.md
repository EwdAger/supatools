# Card 01: Plugin Metadata Contract Upgrade

## Goal

为单一插件身份补齐本地面、远端面、远程同步能力和远端调用契约声明。

## Scope

* 扩展 `plugin.json`
* 增加本地入口、远端入口、远程同步声明、运行模型
* 增加远端 action 列表、输入结构、输出结构声明

## Acceptance

* 插件元数据能明确表达仅本地、仅远程、混合模式
* 平台能基于元数据判断是否可本地运行、是否可入远程仓、是否可远程调用
* 运行模型 `static / oneshot / service` 可被稳定识别

## Suggested Implementation Steps

1. 盘点当前 `plugin.json` 已有字段与实际消费位置
2. 设计第一阶段元数据结构
3. 明确能力组合矩阵
4. 补齐共享类型与元数据解析层
5. 更新本地插件安装链路的元数据持久化
6. 更新内置插件 / 开发态插件 / 市场插件的元数据读取逻辑
7. 定义元数据校验失败策略
8. 增加回归测试样例

## Done Means

* 新的元数据结构已经定稿并可被代码稳定解析
* 三类插件能力组合都能被平台区分
* 本地安装链路、内置插件链路、开发态插件链路都不会丢失新字段
* 后续 Card 2 / Card 11 / Card 12 可以直接消费这套契约
