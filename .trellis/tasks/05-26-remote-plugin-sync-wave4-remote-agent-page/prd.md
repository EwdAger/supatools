# implement: remote plugin sync wave 4 remote agent page

## Goal

把当前远程 Agent 页面重构成小窗友好的列表 -> 二级页结构，并让它承载已经落地的 Wave 2 / Wave 3 数据能力：

- 机器列表作为默认视图
- 新增机器进入二级页
- 机器详情进入二级页
- 机器详情承载安装命令、远端安装状态、机器-插件配置、同步日志

## What I already know

- Wave 2 已完成：
  - 市场加入远程插件仓
  - 远程仓快照更新链路
  - 市场页远程仓状态按钮
- Wave 3 已完成：
  - Linux 侧 `nohup` agent 常驻运行
  - agent 运行时元信息（pid/logPath）
  - 机器安装状态 API
  - 机器-插件配置基础
- 当前页面原型已经确认：
  - 默认列表页
  - 新增机器二级页
  - 机器详情二级页
  - 不在主页面同时铺开创建、列表、详情三层内容
- 当前真实实现文件：
  - `internal-plugins/setting/src/views/RemoteAgentSetting/RemoteAgentSetting.vue`

## Requirements

- 默认主视图只显示机器列表和轻量摘要
- “新增远程机器”进入二级页
- 点击机器进入“机器详情”二级页
- 机器详情页展示：
  - 远程 Agent 在线状态
  - 安装命令
  - 远端已安装插件列表与状态
  - 机器-插件配置编辑
  - 最近同步日志
- 页面必须消费真实的机器安装状态 API，而不是继续只展示本地可部署插件推导结果
- 机器详情页继续保留进入远程插件仓/批量同步主页面的入口

## Acceptance Criteria

- [x] 默认主视图不再同时摊开创建、列表、详情三层内容
- [x] 新增机器流程迁移到二级页
- [x] 机器详情迁移到二级页
- [x] 机器详情使用真实远端安装状态数据
- [x] 机器详情可编辑机器-插件配置并查看同步日志
