# Card 13: Warehouse Snapshot Update Flow

## Goal

让远程仓快照升级成为独立、可审计的操作。

## Scope

* 仓条目“可更新”状态
* 显式更新仓条目版本
* 仓快照替换
* 与本地安装升级解耦

## Acceptance

* 市场新版本出现时，仓条目可感知可更新
* 用户可显式把仓条目推进到新版本
* 本地安装升级不会自动改写远程仓快照

## Suggested Implementation Steps

1. 定义仓条目的版本比较来源
2. 建立仓条目“可更新”判定逻辑
3. 设计“更新远程仓”动作链
4. 明确更新后对服务器的影响表达
5. 确保本地安装升级与仓快照升级解耦
6. 增加快照替换与状态更新测试

## Done Means

* 仓条目版本升级成为显式独立动作
* 仓与本地安装生命周期保持解耦
* 页面能稳定表达仓条目是否可更新
