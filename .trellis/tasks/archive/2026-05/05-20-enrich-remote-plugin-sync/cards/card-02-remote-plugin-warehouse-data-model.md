# Card 02: Remote Plugin Warehouse Data Model

## Goal

建立独立于本地 `plugins` 的远程插件仓模型。

## Scope

* 远程仓条目数据结构
* 包快照引用
* 版本快照字段
* 可更新状态字段
* 与本地 `plugins` 并存关系

## Acceptance

* 同一插件可同时存在于本地 `plugins` 和远程仓
* 远程仓条目不依赖本地插件安装状态
* 远程仓保存独立包快照

## Suggested Implementation Steps

1. 定义远程插件仓条目的最小字段集合
2. 定义远程仓与本地 `plugins` 的关系规则
3. 明确仓快照引用的存储方式
4. 明确仓条目状态字段
5. 定义仓条目的读取和写入入口
6. 设计仓条目与后续服务器维度状态的拼接边界
7. 增加基础数据模型测试

## Done Means

* 远程插件仓成为独立数据集合
* 仓条目最小字段稳定
* 本地插件集合与仓集合边界清楚
* 后续市场入仓、仓页、批量同步都能直接消费这套模型
