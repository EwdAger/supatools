# Card 11: Remote Invocation Platform API

## Goal

提供通用远程调用平台能力，支持混合型插件的本地控制 + 远端执行。

## Scope

* 通用调用口：`machineId + pluginName + action + payload`
* action 白名单校验
* 输入结构校验
* 输出结构校验
* 标准错误模型
* 机器级摘要日志

## Acceptance

* 本地插件可通过平台调用目标机器上的远端动作
* 未安装、离线、超时、执行失败等场景返回标准错误
* 平台前后校验输入输出结构
* 运行时日志可用于排障

## Suggested Implementation Steps

1. 定义通用调用请求结构
2. 定义标准错误集合
3. 增加 action 白名单校验
4. 增加本地输入结构校验
5. 接入机器-插件配置自动注入
6. 接入远端调用转发逻辑
7. 增加输出结构校验
8. 增加机器级摘要日志
9. 增加典型成功/失败路径测试

## Done Means

* 平台能稳定承载混合型插件远程调用
* 输入输出契约和错误模型都可控
* 运行时日志可审计
