# implement: remote plugin sync wave 5 warehouse page and batch sync

## Goal

实现远程插件仓主页面，并把它作为单服务器批量同步的主操作面。

## What I already know

- Wave 1 已完成：
  - 元数据契约
  - 远程插件仓共享模型
  - 可分发规则
- Wave 2 已完成：
  - 市场加入远程插件仓
  - 仓快照更新链路
  - 市场页远程仓状态按钮
- Wave 3 已完成：
  - Linux 侧 agent 常驻运行
  - 机器安装状态 API
  - 机器-插件配置基础
- Wave 4 已完成：
  - 远程 Agent 页面小窗重构
  - 机器详情页可查看远端安装状态、配置、日志
- 已确认的交互结论：
  - 远程插件仓页与单服务器批量同步页合并为同一主页面
  - 未选服务器时先看仓总览
  - 选定服务器后只默认显示可分发条目
  - 通过显式切换查看不可分发条目和原因

## Requirements

- 提供独立菜单页承载远程插件仓
- 未选服务器时，页面展示：
  - 仓总览
  - 仓条目列表
  - 服务器选择器
- 仓总览默认优先展示“有新版本可更新”的条目
- 选定服务器后，页面切换到该服务器视角：
  - 默认只展示可分发条目
  - 展示该服务器上的安装/版本/配置状态
  - 支持选择一组条目执行批量同步
- 提供“查看不可分发条目”的显式切换入口，并展示原因
- 批量同步语义只包含：
  - 安装
  - 升级
  - 配置下发
- 不自动卸载远端多余插件

## Acceptance Criteria

- [ ] 远程插件仓页面可在仓总览与服务器上下文之间切换
- [ ] 未选服务器时能正确展示仓条目和可更新状态
- [ ] 选定服务器后默认只显示可分发条目
- [ ] 可切换查看不可分发条目及原因
- [ ] 可对单台服务器执行批量同步
- [ ] 批量同步只执行安装、升级、配置下发

## Out of Scope

- 多服务器同时批量分发
- 插件详情“同步到远程”入口
- 运行时远端调用 UI
- 自动卸载远端多余插件
